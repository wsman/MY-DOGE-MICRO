# Sprint Snapshot Archive

This directory indexes end-of-sprint or milestone execution snapshots.

Use it when sprint, milestone, or roadmap workflows produce a stable snapshot
that should remain discoverable after active planning changes.

Each record should include:

- Sprint or milestone name
- Date range
- Completed stories or deliverables
- Open risks and carryover
- QA and release evidence links
- Retrospective or milestone-review path

Active planning remains in `production/`. This archive keeps historical
snapshots for later audit and learning.

Use `story-closure-index.md` for individual story completions. Index rows are
keyed by story path; update an existing row for the same story path instead of
adding a duplicate. Sprint and milestone closeout files remain separate archive
records in this directory.
