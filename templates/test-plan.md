## Smoke Test Scope

Critical paths to verify before QA hand-off (run via `/smoke-check`):

1. Game launches to main menu without crash
2. New game / session can be started
3. [Primary mechanic introduced or changed this sprint]
4. [System with regression risk from this sprint's changes]
5. Save / load cycle completes without data loss (if save system exists)
6. Performance is within budget on target hardware

Product critical paths to verify before QA hand-off:

1. API/CLI/web entrypoint starts without crash or missing configuration
2. New user or first-run workflow completes
3. Primary API endpoint, CLI command, or web workflow succeeds
4. Error recovery path gives actionable guidance
5. Migration/import/export path completes without data loss
6. Build/package/docs artifacts are generated and versioned correctly
7. Observability signal exists for critical failures (logs, metrics, trace, or report)
