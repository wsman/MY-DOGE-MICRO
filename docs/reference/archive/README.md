# Reference Archive

This directory preserves historical workflow material that is no longer an
active slash command or primary template, but still contains useful domain
guidance.

## Dual-Domain Preservation Rule

This project supports both game development and general product development.
Do not delete game-specific examples, notes, templates, or extra guidance when
adding product support. If a game-specific passage no longer belongs in an
active product-facing flow, move it into one of these places instead:

- a clearly marked **[游戏专用]** section beside the product guidance
- this reference archive, with a short source note
- a game-specific template such as `game-design-document.md`

Product examples are added alongside game examples. They do not replace
player, gameplay, engine, playtest, Game Feel, or GDD reference material.
