# Eval Storyboard

This page points demo owners to evaluation material without duplicating metrics.

## Storyboard

1. Pick the demo prompt set from [sample_prompts.md](sample_prompts.md).
2. Run the research workflow from
   [research_copilot_demo.md](research_copilot_demo.md).
3. Capture expected evidence and artifact behavior.
4. Run the eval path in [../guides/run-eval.md](../guides/run-eval.md).
5. Check quality links in [../quality/eval-metrics.md](../quality/eval-metrics.md).

## Boundaries

- Demo observations are not release approval.
- External benchmark closure stays under progress and QA evidence.
- Keep screen-reader and live-provider evidence in their existing QA paths.
