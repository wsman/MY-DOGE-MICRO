## API / CLI / Web / Data Findings

### What Worked Well

- Macro CLI blocks early when `DEEPSEEK_API_KEY` is absent; it does not silently send the placeholder key to the OpenAI-compatible client.
- The remediation text is explicit enough for an operator to know the next action: set `DEEPSEEK_API_KEY`.
- API invalid market uses the stable S002-009 envelope: `error.code` + `error.message`.
- Correcting the API request recovers in-place; no backend restart was needed.

### Pain Points

- **FR-3 / Low**: Macro CLI remediation is split across English exception text and a Chinese hint. It is usable, but a single bilingual remediation block would be easier to scan.

### Error Recovery

- Missing key path recovery is documented by the CLI message itself; no real key was required for this test.
- API failure recovery was exercised end to end: invalid `bad` market -> structured HTTP 400 -> corrected `us` scan -> SSE `done`.

### Trust and Adoption Blockers

- None blocking for the selected failure/recovery paths.
