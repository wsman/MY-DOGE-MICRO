## API / CLI / Web / Data Findings

### What Worked Well

- Editable install completed successfully from the repository.
- A fresh `DOGE_DB_DIR` was enough for backend settings to point all DB paths at an isolated data directory.
- Backend and web started cleanly once the local `src` namespace collision was bypassed.
- Empty US scanner path completed with SSE `done` instead of crashing.
- Empty archive/data query returned stable schema and zero rows rather than an error.
- yfinance rate limiting degraded to `None`; it did not corrupt the DB or crash the operator session.

### Pain Points

- **FR-1 / Medium**: Backend direct startup remains fragile on this workstation because another local project exposes a top-level `src` package. The evidence run needed `logs/phase2-first-run/backend_bootstrap.py`.
- **FR-2 / Medium**: First live yfinance ingest could not complete because Yahoo Finance returned `Too Many Requests`. The adapter degraded correctly, but first-run documentation should set operator expectations around rate limits and retry timing.

### Error Recovery

- Scanner fallback path is acceptable: absent `[tdx]` + empty `tdx_path` produced a clear SSE message and terminal `done`.
- yfinance failure degraded safely to zero writes. No DB corruption was observed.

### Trust and Adoption Blockers

- A new operator can start the product and perform an empty first scan, but cannot rely on first live yfinance ingest during Yahoo rate-limit windows.
- The backend startup namespace collision is local-environment-specific but should be documented or avoided through a canonical launch command.
